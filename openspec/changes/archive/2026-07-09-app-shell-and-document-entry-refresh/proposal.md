# Why

当前平台已经有文档库、知识库、数据资产和审查工作台的基本骨架，但登录后的用户区、主题切换、文档上传起点和详情承接还需要进一步统一，才能更像成熟的企业级审查工作台。这个 change 先把平台入口和壳层骨架收束好，为后续标注交互、提示词资产和智能体配置提供稳定承载面。

# What Changes

- 让登录后平台壳层中的用户身份、角色、主题切换和退出动作保持稳定可见。
- 将浅色 / 深色主题切换收敛到统一 token 层，避免组件级各自写样式。
- 强化文档库上传入口，支持拖拽上传和点击选择文件上传，并允许文件名、项目名称作为可选元信息。
- 将文档详情作为独立的全屏工作页打开，使用返回按钮回到文档库，而不是在库页右侧嵌套承载。
- 去掉文档库主路径中的快速预览入口，让主流程聚焦“上传 - 进入详情 - 审查处理”。

# Capabilities

### New Capabilities

本 change 不新增独立业务域，主要是把现有入口体验与全局壳层收束到统一规范。

### Modified Capabilities

- `platform-shell`: 角色区、主题切换、导航层级、详情页承接方式会被统一调整。
- `document-review-task`: 上传入口、任务进入方式、详情页载入方式和预览入口会被统一调整。

# Impact

影响 `src/App.tsx`、`src/styles.css` 以及文档库和详情页的页面组织方式。这个 change 完成后，后续 review popover、模式切换、提示词资产和知识库接入都可以直接挂在同一套壳层和 token 上。
