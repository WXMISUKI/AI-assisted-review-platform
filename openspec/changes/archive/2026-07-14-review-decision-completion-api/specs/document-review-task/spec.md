# document-review-task Delta

## ADDED Requirements

### Requirement: Backend reviewer action endpoints
The backend SHALL expose task-scoped reviewer action endpoints for issue decisions, draft edits, manual issue creation, manual issue deletion, and review completion.

#### Scenario: Client performs a reviewer action
- **WHEN** the frontend submits a reviewer action for a persisted task
- **THEN** the backend mutates that task snapshot and returns the updated task or a safe structured error

### Requirement: Completed task persists backend result asset
The document review task SHALL persist the result asset generated by the backend completion action.

#### Scenario: Review completion succeeds
- **WHEN** the backend completes a review task
- **THEN** the task is marked `completed`, issue counts are refreshed, and the generated result asset is stored on the task aggregate
