# review-workbench Specification

## Purpose

Expose compact issue provenance in the review workbench so reviewers can distinguish generated, fallback, and manual findings.

## Requirements

### Requirement: Compact issue provenance display
The review workbench SHALL display compact provenance labels for review issues when provenance metadata is available.

#### Scenario: Issue was LLM-generated
- **WHEN** an issue has LLM generation provenance
- **THEN** the issue card indicates it was generated by the LLM adapter

#### Scenario: Issue came from deterministic fallback
- **WHEN** an issue has deterministic fallback provenance
- **THEN** the issue card indicates fallback generation without presenting it as a provider failure

#### Scenario: Issue is manual
- **WHEN** an issue source is manual
- **THEN** the issue card remains clearly distinguishable as a manual annotation

### Requirement: Provenance does not disrupt decisions
The workbench SHALL preserve the existing issue decision workflow while displaying provenance.

#### Scenario: User accepts or rejects an issue
- **WHEN** provenance labels are visible
- **THEN** accept, reject, edit, and delete behavior remains unchanged
