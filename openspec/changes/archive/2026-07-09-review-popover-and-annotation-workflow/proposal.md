# Why

当前审查工作台已经具备原文标注、右侧问题卡片和接受/拒绝/修改后接受的基本闭环，但划词后的人工标注入口仍需要更接近成熟审查平台的交互：浮层应该贴近原文、可避让、可拖拽，问题处理动作也应该可反复切换而不是一次点下去就锁死。

# What Changes

- 把划词后出现的人工标注入口明确为可拖拽的浮层，而不是固定在一个容易遮挡正文的位置。
- 让浮层在视口边缘自动避让，保持原文与表单都可读。
- 让接受 / 拒绝按钮变成可高亮、可切换、可恢复的状态，不会因为误触而锁定。
- 为人工新增标注补充删除确认流程，避免误删。
- 收敛审查模式与审查修改模式的差异：审查模式只确认意见，审查修改模式才更新处理后预览。

# Capabilities

### New Capabilities

- `review-popover-workflow`: 支持靠近原文的可拖拽标注浮层和 viewport 避让。

### Modified Capabilities

- `review-workbench`: 划词、标注创建、问题处理、删除确认、模式预览行为都会被统一收束。

# Impact

影响 `src/ReviewWorkbenchPage.tsx`、`src/domain/reviewUtils.ts` 以及与人工标注相关的样式和状态流转。实现后，工作台会更接近成熟的论文查重 / 审查标注平台，也更适合后续接入真实 AI 发现结果。
