# Change Proposal

This change adds a project-level development standards document for both frontend and backend work.

The repository has already grown into a spec-driven, multi-module application. A shared standards document will help keep future changes consistent in structure, security, state management, styling, API contracts, and verification practices.

The standards will be based on the current repo architecture and widely used enterprise practices documented by TypeScript, React, Vite, and OWASP.
