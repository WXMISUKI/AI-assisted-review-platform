# 开工条件核查试点安全边界

本说明适用于 `opening-condition-pilot-controlled-workflow` 变更及后续试点实现。

## 凭据管理

- Dify、OCR、LLM、对象存储和工具服务凭据只能放在服务端环境变量、部署平台 Secret 或受控凭据管理系统中。
- 凭据不得写入仓库文件、OpenSpec 文档、前端环境变量、浏览器响应、任务事件、报告摘要、日志或回调持久化记录。
- 已经出现在外部工作流 YAML、聊天附件、日志或仓库历史中的凭据应视为泄露；若该外部工作流继续用于真实调用，必须由凭据管理员轮换后才能启用。
- 当前开工条件试点默认不启用 Dify 工作流，旧工作流中的凭据不参与平台内可控工作流执行。

## 安全摘要

开工条件试点任务事件、外部工作流导入结果和辅助报告只能保存安全摘要：

- 可以保存：任务状态、阶段、时间、文件名、对象 id、页码或定位摘要、提取值短摘要、置信度、规则结果、人工复核状态。
- 不得保存：密钥、Token、Authorization、Cookie、私有下载链接、预签名 URL、原始 provider trace、完整 prompt、未截断全文、未脱敏 OCR 全文。

## 适配器边界

平台内工作流是试点默认路径。Dify 或其他外部工作流只能作为可选适配器：

- 输入使用工作区、任务、依据版本和对象引用等有界字段。
- 输出必须先归一化为平台记录，再进入页面展示、人工复核或报告归档。
- 外部适配器不可成为依据版本、主数据、核查结论或人工决策的事实数据库。
