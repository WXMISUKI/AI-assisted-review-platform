# Platform-first preflight gates and subcontract knowledge base

Opening-condition review SHALL treat basis confirmation and project master-data initialization as mandatory preflight gates before formal material review. Dify workflow behavior can inform extraction and orchestration design, but Dify Human Input is not part of the main platform maintenance path; the platform owns the review state, human decisions, organization-scoped subcontract knowledge base, and report archive.

## Considered Options

- Keep inserting Dify Human Input into the core review path.
- Use Dify only as a reference or optional adapter and build the maintainable workflow inside the platform.

## Consequences

The next high-value work should focus on platform-owned preflight gating, subcontract-team knowledge-base records, master-data publishing, and material-review task readiness instead of deeper Dify workflow integration.
